# ODI — Marco Industrial 5.0 a 7.0

**Fecha:** 13 Feb 2026
**Propósito:** Documentar la clasificación industrial real de ODI, no como metáfora
sino como implementación concreta de principios evolutivos.

---

## ODI No Es Software. Es Infraestructura Industrial Compartida.

ODI opera como un sistema nervioso económico para pymes colombianas.
No vende licencias. Procesa catálogos como servicio. El negocio accede
al organismo, no compra software.

---

## Industria 5.0 — CUMPLIDO (Operacional)

Los 4 pilares oficiales de Industria 5.0 (Comisión Europea):

### Human-Centric (Centrado en el Humano)

| Principio | Implementación ODI |
|-----------|-------------------|
| No reemplazar humanos | Martha vende en Shopify sin saber qué es una API |
| Cobots digitales | ODI sugiere, el humano decide y cierra |
| Reducción de carga cognitiva | Lead Scoring muestra ALTA/MEDIA/BAJA, no números |
| next_actions para humanos | Cada plan genera acciones concretas para Pereira |

ODI es la extensión digital del experto, no su reemplazo.

### Resilience (Resiliencia)

| Fallo | Respuesta ODI |
|-------|--------------|
| Precios $135 vs $135,000 | → SemanticNormalizer creado |
| Matzu saturado | → Router failover multi-ciudad |
| Redis caído | → Fallback a Postgres |
| Postgres caído | → Fallback a JSON |
| IA principal caída | → Gemini → GPT → Lobotomy |
| Voz ElevenLabs caída | → Operación continúa sin voz |

Resiliencia no es no fallar. Es fallar y fortalecerse.

### Sustainability (Sostenibilidad)

| Métrica | Valor |
|---------|-------|
| Servidor | 1 droplet $24 USD/mes |
| Empresas atendidas | 15 |
| Productos procesados | 15,016 |
| KB docs | 35,816 |
| Equipos IT eliminados | 15 redundancias |
| RAM | 8GB haciendo trabajo enterprise |

Un solo pipeline, un solo objeto canónico (UDM-T) en vez de 10 apps = menos
servidores, menos energía, menos residuo digital. Economía circular digital real.

## Industria 6.0 — EN FRONTERA (Emergente)

ODI ya opera con principios 6.0 aunque no use computación cuántica.

### Autonomía Cognitiva

| Componente | Autonomía |
|------------|-----------|
| Router jerárquico | Decide sin humano qué nodo asignar |
| Tourism Engine | Recalcula viabilidad automáticamente |
| Calendar/HOLD | Bloquea slots sin intervención |
| Failover | Reorganiza red si nodo colapsa |
| Event sourcing | Audita sin que nadie lo pida |

### Cognición Colectiva (ChromaDB)

- 35,816 documentos en KB compartido
- No es indexación simple. Es memoria contextual compartida.
- El costo marginal de "aprender" un repuesto nuevo → cero
- El sistema ya "conoce" la categoría por entrenamiento cruzado

### Autorreparación

Cada error del pipeline fortalece al organismo:
- Error de precios → SemanticNormalizer
- Imágenes incorrectas → Image Matcher mejorado
- Títulos crudos → Normalización automática
- PDFs mal estructurados → Vision Extractor v3

### Lo que falta para 6.0 completo

- Optimización cuántica real
- Gemelos digitales físicos
- Integración IoB (Internet of Bodies)
- Blockchain de trazabilidad médica
- Auto-reconfiguración estructural sin humano

**Veredicto:** ODI está en la frontera entre 5.0 avanzada y 6.0 emergente.

## Industria 7.0 — EN SEMILLA (Potencial Estructural)

Industria 7.0 es teórica pero define: autonomía cognitiva total,
auto-regulación ética, regeneración ambiental activa, economía simbiótica.

### Rasgos 7.0 que ODI ya tiene

**Autonomía Cognitiva Modular:**
Router → Engine → Calendar → Fallback → Eventos.
Todo sin humano. El humano es co-piloto, no operador.

**Regeneración Económica:**
Si un nodo colapsa: no se pierde el paciente, se redistribuye ingreso,
se protege al médico, se protege al paciente.

**Ética Operativa Embebida:**
- No sobrecarga médicos (SLA + capacity limits)
- No vende si no hay capacidad (HOLD con expiración)
- No promete si no puede cumplir (viability check)
- Explica decisiones (reasons en scoring, audit trail)

**Servitización:**
Transforma CAPEX (comprar software) en OPEX (servicio de procesamiento).
Permite que microempresa en Pereira compita con gigantes globales.

### Lo que convertiría a ODI en 7.0 real

**Capa 1: Scoring de Impacto Ambiental**
```
Score_7_0 = Salud + Logística + Margen + Huella_Carbono_Neta
```
Penalizar vuelos innecesarios. Priorizar cercanía geográfica.

**Capa 2: Índice de Bienestar Médico**
- Horas trabajadas por semana del doctor
- Estrés operativo (inferido por saturación)
- Limitar automáticamente carga cuando bienestar baja

**Capa 3: Regeneración Sistémica**
- Si Pereira recibe demasiado flujo → activar incubación de nuevo nodo
- Activar formación educativa (diplomados)
- Redistribuir conocimiento entre nodos

## Scoring de Conversión S_{5.0}

Fórmula refinada con factor de saturación:

```
S_{5.0} = ω₁(Sₘ) + ω₂(Wₛ) + ω₃(Uₐ) + ω₄(Lf) - ω₅(S_SLA)

Donde:
  Sₘ   = Semantic Match (procedimiento ↔ nodo)
  Wₛ   = Sentiment (intención del lead)
  Uₐ   = Urgency (ventana temporal)
  Lf   = Logistics Feasibility (vuelos + hospedaje + viabilidad)
  S_SLA = Saturación del nodo (penalización)
```

Si sentimiento es alto pero saturación es crítica → el sistema prioriza
la red colaborativa en vez de forzar el nodo principal.
Eso es Human-centric: proteger la experiencia del paciente.
